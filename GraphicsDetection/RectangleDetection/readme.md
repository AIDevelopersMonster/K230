# Обнаружение прямоугольников (Rectangle Detection)

## 📋 Описание

Данный проект демонстрирует работу обнаружения прямоугольников на плате **Yahboom K230** с использованием камеры и выводом результатов на LCD-экран.

Скрипт захватывает изображение с камеры, находит на нём прямоугольные объекты с помощью алгоритма `find_rects()`, и отображает результаты на экране с визуальной подсветкой:
- Границы прямоугольников (голубая рамка)
- Углы прямоугольников (круги в вершинах)
- Нумерация найденных объектов
- Статистика (количество прямоугольников и FPS)

## 🛠️ Используемые компоненты

| Компонент | Описание |
|-----------|----------|
| **Sensor** | Камера для захвата изображения |
| **Display (ST7701)** | LCD экран для отображения результатов |
| **MediaManager** | Менеджер медиа для управления ресурсами |

## ⚙️ Настройки

В начале скрипта можно изменить следующие параметры:

| Параметр | Значение по умолчанию | Описание |
|----------|----------------------|----------|
| `PICTURE_WIDTH` | 400 | Ширина захватываемого изображения |
| `PICTURE_HEIGHT` | 240 | Высота захватываемого изображения |
| `DISPLAY_WIDTH` | 640 | Ширина LCD экрана |
| `DISPLAY_HEIGHT` | 480 | Высота LCD экрана |
| `RECT_THRESHOLD` | 8000 | Порог обнаружения (чем выше, тем более контрастные прямоугольники будут найдены) |

## 🚀 Запуск

1. Подключите плату Yahboom K230 к компьютеру
2. Откройте CanMV IDE
3. Загрузите скрипт `01_rectangle_detection_lcd_demo.py` на плату
4. Запустите скрипт через IDE

## 📝 Как это работает

1. **Инициализация**: Скрипт инициализирует камеру и LCD дисплей
2. **Захват кадра**: Камера делает снимок текущего изображения
3. **Поиск прямоугольников**: Алгоритм `find_rects()` анализирует изображение и находит прямоугольные объекты
4. **Отрисовка**: Найденные прямоугольники подсвечиваются на изображении
5. **Вывод**: Изображение отображается на LCD экране
6. **Повтор**: Цикл повторяется для следующего кадра

## 🔧 Настройка порога обнаружения

Параметр `RECT_THRESHOLD` контролирует чувствительность обнаружения:
- **Низкое значение** (например, 3000): Будет находить больше прямоугольников, включая менее контрастные
- **Высокое значение** (например, 15000): Будет находить только самые контрастные и чёткие прямоугольники

Подберите значение под ваши условия освещения и задачи.

## 📷 Пример работы

На экране вы увидите:
- Вид с камеры в центре экрана
- Голубые рамки вокруг найденных прямоугольников
- Кружки в углах каждого прямоугольника
- Номер каждого найденного объекта (RECT 1, RECT 2, ...)
- Статистику в левом верхнем углу (количество прямоугольников и FPS)

## ❓ Возможные проблемы

| Проблема | Решение |
|----------|---------|
| Не находит прямоугольники | Уменьшите `RECT_THRESHOLD` или улучшите освещение |
| Находит слишком много шума | Увеличьте `RECT_THRESHOLD` |
| Низкий FPS | Уменьшите разрешение камеры (`PICTURE_WIDTH`/`PICTURE_HEIGHT`) |
