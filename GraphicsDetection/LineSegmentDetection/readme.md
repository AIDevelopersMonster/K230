# Детекция отрезков линий (K230)

## Описание

Пример показывает, как находить отрезки линий с помощью `find_line_segments()` на Yahboom K230 / CanMV.

В папке есть три типа примеров:

1. работа с камерой в реальном времени;
2. работа с синтетическим изображением в памяти;
3. работа с заранее подготовленными картинками из папки `img`.

Функция `find_line_segments()` использует алгоритм LSD (Line Segment Detector) и/или преобразование Хафа. Она возвращает список объектов `image.line`, каждый из которых содержит координаты начала и конца отрезка. Параметры `merge_distance` и `max_theta_diff` управляют объединением близких линий и линий с похожим углом наклона.

## Какой скрипт с какой картинкой

| Скрипт | Источник | Что показывает |
|---|---|---|
| `01_line_segment_camera.py` | камера | Поиск всех линий в реальном времени с камеры |
| `02_line_segment_static.py` | память | Учебный пример: рисует линии в памяти и ищет их |
| `03_line_segment_from_images.py` | файл PNG | Универсальный пример: можно искать все, только горизонтальные или только вертикальные линии |
| `04_detect_horizontal_lines.py` | `img/horizontal_lines.png` | Ищет только горизонтальные линии на статичной картинке |
| `05_detect_vertical_lines.py` | `img/vertical_lines.png` | Ищет только вертикальные линии на статичной картинке |
| `06_detect_all_lines_overlay.py` | `img/mixed_lines.png` | Ищет все линии и раскрашивает: горизонтальные, вертикальные, наклонные |
| `07_camera_horizontal_target.py` | камера | Поиск ТОЛЬКО горизонтальных линий в реальном времени |
| `08_camera_vertical_target.py` | камера | Поиск ТОЛЬКО вертикальных линий в реальном времени |
| `09_camera_mixed_classification.py` | камера | Классификация всех линий (горизонтальные/вертикальные/наклонные) в реальном времени |

**Цвета в примерах:**

- **зелёный** — горизонтальные линии;
- **красный** — вертикальные линии;
- **синий** — наклонные линии;
- **жёлтый текст** — количество найденных линий.

## Куда положить картинки

На компьютере картинки лежат в репозитории:

```text
Graphics detection/Line segment detection/img/
```

На плате K230 скопируйте их на SD-карту в такой же путь:

```text
/sdcard/Graphics detection/Line segment detection/img/
```

Если путь другой, измените строку `IMAGE_PATH` в нужном скрипте:

```python
IMAGE_PATH = "/sdcard/ваш_путь/img/vertical_horizontal_lines.png"
```

## Как запустить

### Примеры с камерой

1. Подключите камеру к K230
2. Запустите один из скриптов:
   - `01_line_segment_camera.py` — поиск всех линий
   - `07_camera_horizontal_target.py` — только горизонтальные
   - `08_camera_vertical_target.py` — только вертикальные
   - `09_camera_mixed_classification.py` — классификация всех типов
3. Наведите камеру на объект с чёткими линиями

### Примеры с картинками

1. Скопируйте папку `img` на SD-карту K230
2. Откройте скрипт в CanMV IDE
3. Проверьте переменную `IMAGE_PATH`
4. Запустите скрипт

### Статический пример

Запустите `02_line_segment_static.py` — он не требует ни камеры, ни SD-карты.

В `03_line_segment_from_images.py` можно менять режим поиска:

```python
DETECT_MODE = "all"        # искать все линии

```

Также можно менять картинку:

```python
IMAGE_PATH = "/sdcard/Graphics detection/Line segment detection/img/mixed_lines.png"
```

## Основной API

```python
lines = img.find_line_segments(merge_distance=15, max_theta_diff=10)
```

**Параметры:**

- `merge_distance` — максимальное расстояние между двумя отрезками, при котором они могут быть объединены (в пикселях);
- `max_theta_diff` — максимальная разница углов для объединения отрезков (в градусах).

**Результат:**

- список объектов линий;
- координаты линии можно получить через `line.line()` → `(x1, y1, x2, y2)`.

**Пример использования:**

```python
for line in lines:
    coords = line.line()
    img.draw_line(coords, color=(0, 255, 0), thickness=3)
```

## Если картинка не открывается

Проверьте:

1. файл действительно лежит на SD-карте;
2. имя файла совпадает с `IMAGE_PATH`;
3. расширение `.png` написано правильно;
4. путь не содержит лишних пробелов;
5. картинка имеет понятный формат, лучше `PNG`, `640x480`.

---

## Если линии не находятся

Попробуйте:

- сделать линии толще: 3–6 пикселей;
- использовать белые линии на чёрном фоне;
- увеличить `merge_distance`;
- увеличить `ANGLE_TOLERANCE_DEG` для фильтрации горизонтальных/вертикальных линий;
- проверить, что линии не касаются края изображения;
- обеспечить хорошее освещение при работе с камерой;
- убрать блики и отражения.

---

## Советы для работы с камерой

1. **Освещение**: обеспечьте равномерное освещение объекта
2. **Контраст**: используйте объекты с высоким контрастом
3. **Расстояние**: подберите оптимальное расстояние до объекта
4. **Угол**: держите камеру перпендикулярно плоскости объекта
5. **Стабильность**: закрепите камеру для стабильного изображения
