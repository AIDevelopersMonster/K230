# Linear Regression (K230)

## 📌 Описание

Модуль демонстрирует использование функции линейной регрессии `img.get_regression()` для поиска и отслеживания линии на изображении с камеры. Это основа для создания роботов, следующих по линии (line following robots).

📖 Основано на примере из документации K230 (line patrol) — камера смотрит на чёрную линию на белом фоне.

---

## 🚀 Быстрый старт

### Требования
- Плата Yahboom K230
- Камера, подключённая к K230
- Дисплей ST7701 (опционально, для визуализации)

### Установка
1. Скопируйте файлы `.py` на SD-карту или во внутреннюю память K230
2. Подключите камеру к плате
3. Запустите нужный скрипт через IDE или напрямую на устройстве

## 🧪 Примеры

| Файл | Описание | Для кого |
|------|----------|----------|
| `01_linear_regression_camera.py` | Базовый поиск линии с камеры | Новички |
| `02_linear_regression_binary.py` | Поиск линии с предварительной бинаризацией | Продвинутые |
| `03_line_following_pid.py` | Расчёт ошибки для управления роботом | Робототехника |

### Краткое описание скриптов

#### 01_linear_regression_camera.py
Базовый пример:
- Получает изображение с камеры в оттенках серого
- Ищет тёмную линию методом линейной регрессии
- Рисует найденную линию на экране
- Показывает "магнитуду" (качество обнаружения)

**Что такое магнитуда?**  
Чем больше значение, тем лучше видна линия и тем больше пикселей попало в область обнаружения.

#### 02_linear_regression_binary.py
Улучшенная версия:
- Применяет бинаризацию перед поиском линии
- Все пиксели делятся на чёрные (0) и белые (255)
- Повышает стабильность при плохом освещении
- Убирает шумы и лишние детали

**Зачем нужна бинаризация?**  
Она упрощает изображение, оставляя только важные контуры. Это как сделать ксерокопию документа — остаётся только чёрное и белое.

#### 03_line_following_pid.py
Для робототехники:
- Находит линию и вычисляет её центр
- Сравнивает с центром экрана
- Выдаёт ошибку (error) — отклонение от центра
- Готово для использования в PID-регуляторе

**Как использовать ошибку?**
- `error > 0` — линия справа, поверни вправо
- `error < 0` — линия слева, поверни влево
- `error = 0` — линия по центру, едь прямо

## 🧠 Как работает линейная регрессия

### Алгоритм работы

1. **Получение изображения**  
   Камера снимает кадр в оттенках серого (640x480 пикселей)

2. **Фильтрация по порогу (Threshold)**  
   Выбираем пиксели в диапазоне яркости (например, 0-100 для тёмных)

3. **Поиск области**  
   Находим связную область из подходящих пикселей

4. **Метод наименьших квадратов**  
   Строим прямую линию, которая наилучшим образом проходит через найденные пиксели

5. **Результат**  
   Получаем координаты начала (x1, y1) и конца (x2, y2) линии

### Визуализация

```
Камера → [Изображение] → [Порог] → [Бинаризация] → [Регрессия] → Линия
                                                      ↓
                                            (x1,y1) ─────── (x2,y2)
```

## ⚙️ Настройка параметров

### Порог (THRESHOLD)

```python
THRESHOLD = (0, 100)  # Для чёрной линии на белом фоне
THRESHOLD = (150, 255)  # Для белой линии на чёрном фоне
```

**Как подобрать порог:**
1. Запустите скрипт
2. Посмотрите на изображение с камеры
3. Если линия не находится — увеличьте диапазон
4. Если находится мусор — уменьшите диапазон

### Разрешение

```python
sensor.set_framesize(width=640, height=480)  # VGA
sensor.set_framesize(width=320, height=240)  # QVGA (быстрее)
```

**Влияние разрешения:**
- Больше разрешение → выше точность, но медленнее обработка
- Меньше разрешение → ниже точность, но выше FPS

## 🤖 Применение для робота

### Простая логика управления

```python
if line:
    x_center = (line.x1() + line.x2()) / 2
    error = x_center - 320  # 320 = половина ширины экрана
    
    if error > 20:
        turn_right()    # Линия справа
    elif error < -20:
        turn_left()     # Линия слева
    else:
        go_forward()    # Линия по центру
```

### PID-регулятор (продвинутый уровень)

```python
Kp = 0.5  # Пропорциональный коэффициент
Ki = 0.01 # Интегральный коэффициент
Kd = 0.1  # Дифференциальный коэффициент

integral = 0
previous_error = 0

while True:
    error = get_error()  # Получить ошибку от линии
    
    integral += error
    derivative = error - previous_error
    
    output = Kp * error + Ki * integral + Kd * derivative
    
    set_motor_speed(output)
    previous_error = error
```

## 🔧 Решение проблем

### Линия не находится
- Проверьте освещение (должно быть равномерным)
- Adjust THRESHOLD под ваши условия
- Убедитесь, что линия контрастная

### Низкий FPS
- Уменьшите разрешение камеры
- Отключите дисплей (если не нужен)
- Используйте бинаризацию (быстрее обработка)

### Линия "дрожит"
- Добавьте усреднение по нескольким кадрам
- Используйте PID-регулятор вместо простой логики
- Проверьте крепление камеры (должна быть жёсткой)
