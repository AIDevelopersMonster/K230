# K230: Многоцветное распознавание

Эта папка содержит демонстрационные примеры для **Yahboom K230 Vision Module**: камера одновременно ищет несколько цветов в кадре, рисует рамки вокруг найденных областей, подписывает цвет и показывает FPS.

Тема основана на уроке **Multicolor Recognition**: после базового распознавания одного цвета программа по очереди применяет несколько LAB-порогов к одному кадру. Так можно одновременно находить красные, зеленые и синие объекты.

## Файлы

| Файл | Назначение |
| --- | --- |
| `01_multicolor_recognition.py` | Базовый пример: распознавание RED, GREEN, BLUE с камеры. |
| `02_multicolor_uart.py` | Распознавание нескольких цветов + отправка данных по UART через Yahboom protocol. |
| `03_multicolor_file_io.py` | Распознавание нескольких цветов + чтение LAB-порогов из файла и запись результата в файл. |
| `readme.md` | Инструкция на русском языке. |
| `readme_en.md` | Instruction in English. |

## Быстрый старт

1. Откройте `01_multicolor_recognition.py` в CanMV IDE.
2. Подключите K230 к компьютеру.
3. Наведите камеру на красный, зеленый или синий объект.
4. Запустите скрипт.
5. На экране появятся рамки, подписи `RED`, `GREEN`, `BLUE`, FPS и счетчики по цветам.

## Как работает распознавание нескольких цветов

Для каждого цвета задается отдельный LAB-порог:

```python
COLOR_THRESHOLDS = [
    (0, 66, 7, 127, 3, 127),          # RED
    (42, 100, -128, -17, 6, 66),      # GREEN
    (43, 99, -43, -4, -56, -7),       # BLUE
]
```

Формат каждого порога:

```python
(L_min, L_max, A_min, A_max, B_min, B_max)
```

В основном цикле программа получает кадр и проверяет его для каждого цвета:

```python
for i in range(len(COLOR_THRESHOLDS)):
    counts[i] = process_color(img, i)
```

Внутри `process_color()` вызывается:

```python
blobs = img.find_blobs([threshold], pixels_threshold=50, area_threshold=500, merge=True)
```

После этого для каждого найденного объекта рисуется рамка, крестик в центре и текстовая подпись цвета.

## Пример с UART

Файл `02_multicolor_uart.py` показывает, как отправлять координаты найденного объекта через UART:

```python
pto_data = pto.get_multi_color_data(x, y, w, h, label)
uart.send(pto_data)
```

Используется скорость:

```python
YbUart(baudrate=115200)
```

Если в вашей прошивке нет `libs.YbProtocol` или `ybUtils.YbUart`, пример не падает, а отключает UART и продолжает распознавание на экране.

## Чтение файлов на K230

Файл `03_multicolor_file_io.py` показывает чтение LAB-порогов из текстового файла.

Имя файла:

```text
multicolor_thresholds.txt
```

Пример содержимого:

```text
RED:0,66,7,127,3,127
GREEN:42,100,-128,-17,6,66
BLUE:43,99,-43,-4,-56,-7
```

Чтение файла:

```python
with open("/sdcard/multicolor_thresholds.txt", "r") as f:
    text = f.read()
```

Разбор строки:

```python
label, values_text = line.split(":", 1)
parts = values_text.replace(" ", "").split(",")
threshold = tuple([int(v) for v in parts])
```

На CanMV/K230 относительный путь иногда недоступен для записи при запуске из IDE. Поэтому пример сначала ищет доступную папку:

```python
FILE_DIR_CANDIDATES = ["/sdcard", "/data", "/flash", "/"]
```

Если ни одна папка не доступна для записи, пример продолжает работать с порогами по умолчанию.

## Запись файлов на K230

Запись выполняется через `open(..., "w")`:

```python
with open("/sdcard/multicolor_result.txt", "w") as f:
    f.write("RED=1\nGREEN=0\nBLUE=2\n")
```

Режимы открытия:

| Режим | Что делает |
| --- | --- |
| `"r"` | Чтение существующего файла. |
| `"w"` | Запись с заменой старого содержимого. Если файла нет, он будет создан. |
| `"a"` | Добавление текста в конец файла. |

Файл результата `multicolor_result.txt` обновляется примерно раз в секунду:

```text
fps=62.500
time_ms=123456
RED=1
GREEN=0
BLUE=2
total=3
```

Почему не записывать каждый кадр: камера работает быстро, и постоянная запись в файловую систему может быть лишней. Лучше записывать результат периодически или только при изменении данных.

## Настройка точности

Если цвет распознается плохо:

1. Улучшите освещение.
2. Уберите блики и тени.
3. Используйте CanMV Threshold Editor для подбора LAB-порогов.
4. Увеличьте `pixels_threshold` и `area_threshold`, если появляется шум.
5. Уменьшите `area_threshold`, если маленькие объекты не находятся.
6. Проверьте, что объект достаточно отличается от фона.

## Структура программы

1. Импорт модулей.
2. Определение размеров экрана и LAB-порогов.
3. Инициализация камеры и дисплея.
4. Основной цикл:
   - получить кадр;
   - пройти по списку цветов;
   - найти цветовые области через `find_blobs()`;
   - нарисовать рамки, крестики и подписи;
   - вывести FPS и счетчики;
   - показать изображение.
5. Обработка ошибок и освобождение ресурсов.

## Как добавить свой цвет

Добавьте новый LAB-порог, цвет рамки и подпись:

```python
COLOR_THRESHOLDS.append((37, 100, -128, 127, -128, -27))
DRAW_COLORS.append((0, 255, 255))
COLOR_LABELS.append("CYAN")
```

В `03_multicolor_file_io.py` можно просто добавить строку в `multicolor_thresholds.txt`:

```text
CYAN:37,100,-128,127,-128,-27
```
