# K230: Подсчет объектов по цвету

Эта папка содержит демонстрационные примеры для Yahboom K230 Vision Module: камера ищет объекты одного цвета, обводит их рамкой и выводит количество найденных объектов на экран.

Материал подготовлен по уроку **Object counting**: в нем предлагается использовать инструмент **Generate Random Number of Coins**, сгенерировать картинку с монетами, запустить пример и навести камеру на изображение. В левом верхнем углу экрана отображаются FPS и число найденных монет.

## Файлы

| Файл | Назначение |
| --- | --- |
| `01_object_counting_coins.py` | Базовый пример подсчета объектов по LAB-порогу. |
| `02_object_counting_file_io.py` | Пример подсчета объектов + чтение LAB-порога из файла и запись результата в файл. |
| `readme.md` | Инструкция на русском языке. |
| `readme_en.md` | Instruction in English. |

## Быстрый старт

1. Откройте `01_object_counting_coins.py` в CanMV IDE.
2. Подключите K230 к компьютеру.
3. Сгенерируйте изображение с монетами в инструменте Yahboom **Generate Random Number of Coins**.
4. Запустите скрипт и наведите камеру на изображение.
5. На экране появятся рамки вокруг найденных объектов, FPS и `Num` — количество объектов.

## Как работает подсчет

Основная функция поиска:

```python
blobs = img.find_blobs([threshold], pixels_threshold=50, area_threshold=50, merge=True)
```

`threshold` — это LAB-порог цвета в формате:

```python
(L_min, L_max, A_min, A_max, B_min, B_max)
```

В примере для монет используется:

```python
TRACK_THRESHOLD = (0, 100, -7, 127, 10, 83)
```

Дальше программа проходит по всем найденным областям:

```python
for blob in blobs:
    img.draw_rectangle(blob[0:4])
    img.draw_cross(blob[5], blob[6])
```

Количество объектов — это длина списка `blobs`:

```python
count = len(blobs)
```

## Чтение файлов на K230

Во втором примере `02_object_counting_file_io.py` показано чтение текстового файла с порогом цвета.

Файл:

```text
object_threshold.txt
```

Содержимое файла должно быть одной строкой из 6 чисел:

```text
0,100,-7,127,10,83
```

Чтение файла:

```python
with open("object_threshold.txt", "r") as f:
    text = f.read()
```

Преобразование текста в LAB-порог:

```python
parts = text.strip().replace(" ", "").split(",")
threshold = tuple([int(value) for value in parts])
```

Если файл отсутствует, пример сам создает его со значением по умолчанию.

## Запись файлов на K230

Запись текстового файла выполняется через `open(..., "w")`:

```python
with open("object_count_result.txt", "w") as f:
    f.write("objects=24\n")
```

Режимы открытия:

| Режим | Что делает |
| --- | --- |
| `"r"` | Чтение существующего файла. |
| `"w"` | Запись с заменой старого содержимого. Если файла нет, он будет создан. |
| `"a"` | Добавление текста в конец файла. |

Во втором примере результат записывается в файл `object_count_result.txt` примерно раз в секунду:

```text
objects=24
fps=65.529
time_ms=123456
```

Почему не записывать каждый кадр: камера может работать с высокой частотой, и постоянная запись в файловую систему не нужна. Лучше записывать результат периодически или только при изменении счетчика.

## Настройка точности

Если распознавание плохое:

1. Улучшите освещение.
2. Уберите блики на экране или объекте.
3. Настройте LAB-порог под реальные условия.
4. Увеличьте `pixels_threshold` и `area_threshold`, если появляется шум.
5. Уменьшите пороги, если маленькие объекты не находятся.

## Структура программы

1. Импорт модулей.
2. Определение констант: размер экрана, LAB-порог, цвета текста.
3. Инициализация камеры и дисплея.
4. Основной цикл:
   - получить кадр;
   - найти цветовые области;
   - нарисовать рамки и крестики;
   - вывести FPS и количество;
   - показать изображение.
5. Обработка ошибок и освобождение ресурсов.

## Полезные изменения

Чтобы считать другой цвет, замените `TRACK_THRESHOLD` или содержимое `object_threshold.txt`.

Чтобы сохранить историю измерений, замените режим записи результата с `"w"` на `"a"`:

```python
with open("object_count_result.txt", "a") as f:
    f.write("objects=%d\n" % count)
```

Так новые строки будут добавляться в конец файла, а не заменять старые данные.
