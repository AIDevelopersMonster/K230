# K230: Color Line Patrol

Эта папка содержит демонстрационные примеры для **Yahboom K230 Vision Module** по теме цветного движения по линии: камера ищет линию заданного цвета в нижней части кадра, вычисляет отклонение линии от центра экрана и, в расширенном примере, отправляет команды управления моторами через UART.

Основная идея: линия распознается как цветовой блок через `img.find_blobs()`. Из найденных областей выбирается самая большая, затем вычисляется ее центр. Если центр линии левее или правее центра экрана, PID-регулятор рассчитывает разные скорости для левого и правого мотора.

## Файлы

| Файл | Назначение |
| --- | --- |
| `01_color_line_patrol.py` | Безопасная визуальная демонстрация: поиск линии, рамка, центр линии, ошибка отклонения. UART и моторы не используются. |
| `02_color_line_patrol_uart_pid.py` | Демонстрация line patrol с UART и PID: отправка команд `$left,right#`. |
| `03_color_line_patrol_file_io.py` | Демонстрация чтения настроек из файла и записи результата в файл. |
| `readme.md` | Инструкция на русском языке. |
| `readme_en.md` | Instruction in English. |

## Быстрый старт

1. Откройте `01_color_line_patrol.py` в CanMV IDE.
2. Подключите K230 к компьютеру.
3. Подготовьте линию на светлом фоне: например, зеленую или черную полоску.
4. Наведите камеру так, чтобы линия была в нижней половине кадра.
5. Запустите скрипт.
6. На экране появятся:
   - серая рамка ROI — область поиска линии;
   - зеленая вертикальная линия — центр экрана;
   - красная вертикальная линия — текущий центр найденной линии;
   - рамка вокруг найденной линии;
   - ошибка `error`.

## Как работает поиск линии

В примерах используются LAB-пороги:

```python
THRESHOLDS = [
    (21, 33, -15, 9, -9, 6),       # BLACK
    (40, 86, -44, -20, -24, 25),   # GREEN
]
```

Формат порога:

```python
(L_min, L_max, A_min, A_max, B_min, B_max)
```

Поиск выполняется только в нижней половине кадра:

```python
ROI = (0, DISPLAY_HEIGHT // 2, DISPLAY_WIDTH, DISPLAY_HEIGHT // 2)
```

Это удобно для робота, который смотрит вперед и вниз: линия обычно находится ближе к нижней части изображения.

Основной вызов поиска:

```python
blobs = img.find_blobs([threshold], roi=ROI, pixels_threshold=80, area_threshold=80, merge=True)
```

Из всех найденных областей выбирается самая большая:

```python
largest_blob = max(blobs, key=lambda b: b[4])
```

Дальше вычисляется центр найденной линии:

```python
current_x = x + w // 2
error = SCREEN_CENTER - current_x
```

Если `error` близок к нулю, линия находится около центра. Если `error` положительный или отрицательный, линия смещена влево или вправо.

## PID и UART

Файл `02_color_line_patrol_uart_pid.py` показывает управление через PID.

PID-параметры:

```python
KP = 1.0
KI = 0.1
KD = 0.2
BASE_SPEED = 300
```

Расчет скоростей:

```python
output = KP * error + KI * integral + KD * derivative
left_speed = BASE_SPEED - output
right_speed = BASE_SPEED + output
```

Команда отправляется по UART в формате:

```text
$left,right#
```

Например:

```text
$280,320#
```

Если линия не найдена, отправляется команда остановки:

```text
$0,0#
```

UART инициализируется так:

```python
uart = YbUart(9600)
```

Если библиотека `ybUtils.YbUart` отсутствует в прошивке, пример не падает, а отключает UART и продолжает показывать визуальную отладку на экране.

## Чтение файлов на K230

Файл `03_color_line_patrol_file_io.py` показывает чтение настроек из текстового файла.

Имя файла:

```text
line_patrol_config.txt
```

Пример содержимого:

```text
# K230 Color Line Patrol config
COLOR=GREEN
THRESHOLD=40,86,-44,-20,-24,25
KP=1.0
KI=0.1
KD=0.2
BASE_SPEED=300
```

Чтение файла:

```python
with open("/sdcard/line_patrol_config.txt", "r") as f:
    text = f.read()
```

Разбор строки:

```python
key, value = line.split("=", 1)
```

Разбор LAB-порога:

```python
parts = value.replace(" ", "").split(",")
threshold = tuple([int(v) for v in parts])
```

На CanMV/K230 относительный путь иногда недоступен для записи при запуске из IDE. Поэтому пример сначала ищет доступную папку:

```python
FILE_DIR_CANDIDATES = ["/sdcard", "/data", "/flash", "/"]
```

Если файловая система недоступна для записи, пример продолжает работать со значениями по умолчанию.

## Запись файлов на K230

Запись выполняется через `open(..., "w")`:

```python
with open("/sdcard/line_patrol_result.txt", "w") as f:
    f.write("error=0\n")
```

Режимы открытия:

| Режим | Что делает |
| --- | --- |
| `"r"` | Чтение существующего файла. |
| `"w"` | Запись с заменой старого содержимого. Если файла нет, он будет создан. |
| `"a"` | Добавление текста в конец файла. |

Файл результата `line_patrol_result.txt` обновляется примерно раз в секунду:

```text
time_ms=123456
fps=55.300
found=True
color=GREEN
error=-12
left_speed=312
right_speed=288
```

Почему не записывать каждый кадр: камера работает быстро, а постоянная запись в файловую систему может быть лишней. Лучше записывать результат периодически.

## Настройка точности

Если линия распознается плохо:

1. Улучшите освещение.
2. Уберите блики и тени.
3. Сделайте фон контрастным относительно линии.
4. Настройте LAB-порог через CanMV Threshold Editor.
5. Увеличьте `pixels_threshold` и `area_threshold`, если появляется шум.
6. Уменьшите `area_threshold`, если тонкая линия не находится.
7. Проверьте ROI: линия должна попадать в нижнюю половину кадра.

## Настройка PID

Если робот слишком резко поворачивает, уменьшите `KP`.

Если робот медленно возвращается к линии, увеличьте `KP`.

Если есть постоянное смещение, аккуратно увеличьте `KI`.

Если движение дергается, измените `KD` или уменьшите скорость.

Начинайте тест с небольших значений скорости и проверяйте команды без моторов или на поднятом роботе.

## Как выбрать другой цвет линии

В `01_color_line_patrol.py` и `02_color_line_patrol_uart_pid.py` измените:

```python
COLOR_INDEX = 0  # BLACK
COLOR_INDEX = 1  # GREEN
```

В `03_color_line_patrol_file_io.py` измените строку в `line_patrol_config.txt`:

```text
COLOR=BLACK
```

или задайте свой LAB-порог:

```text
THRESHOLD=21,33,-15,9,-9,6
```
