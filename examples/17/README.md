# Touch Display на K230

## Что это такое

Touch Display — это сенсорный экран, который реагирует на касания пользователя.

Это позволяет:
- создавать графические интерфейсы (GUI)
- управлять устройством пальцем или стилусом
- делать интерактивные приложения с кнопками и элементами управления
- реализовывать рисование и жесты

## Основная идея

```text
палец → координаты (x, y) → обработка → действие
```

Сенсорный экран постоянно сканируется на наличие касаний. Когда вы касаетесь экрана, 
контроллер возвращает координаты точки касания, которые можно использовать для:
- определения нажатия кнопки
- рисования линий и фигур
- перемещения объектов
- выбора элементов меню

## Основные шаги работы

1. **Инициализация touch** - `TOUCH(0)` - подготовка сенсорного контроллера
2. **Чтение координат** - `tp.read(1)` - получение текущих координат касания
3. **Обработка** - проверка координат, определение действий
4. **Отображение результата** - обновление экрана с учётом ввода

## API и функции

### machine.TOUCH(0)
Инициализирует сенсорный контроллер. Создаёт объект для работы с touch-экраном.
Для Yahboom K230 используется `TOUCH(0)`.

### tp.read(n)
Возвращает список точек касания (максимум n точек).
Каждая точка содержит:
- `x` - горизонтальная координата (0..ширина_экрана-1)
- `y` - вертикальная координата (0..высота_экрана-1)
- `event` - тип события:
  - `0` или `TOUCH.EVENT_DOWN` - начало касания (палец коснулся экрана)
  - `1` или `TOUCH.EVENT_MOVE` - движение пальца по экрану
  - `2` или `TOUCH.EVENT_UP` - конец касания (палец убран с экрана)

## Как это работает (для новичков)

### Структура типичной программы:

```python
# 1. Импорт библиотек
from media.display import *
from media.media import *
from machine import TOUCH
import image

# 2. Инициализация
DISPLAY_WIDTH = 640
DISPLAY_HEIGHT = 480
tp = TOUCH(0)  # Создаём объект touch-контроллера
Display.init(Display.ST7701, width=DISPLAY_WIDTH, height=DISPLAY_HEIGHT, to_ide=True)
MediaManager.init()

# 3. Создание изображения (буфера)
img = image.Image(DISPLAY_WIDTH, DISPLAY_HEIGHT, image.ARGB8888)
img.clear()

# 4. Главный цикл
while True:
    os.exitpoint()  # Проверяем флаг выхода
    
    # Очищаем буфер
    img.clear()
    
    # Читаем касание
    points = tp.read(1)
    
    # Если есть касание - обрабатываем
    if len(points):
        pt = points[0]
        x, y = pt.x, pt.y
        print(f"Касание: x={x}, y={y}")
        # ... ваша логика ...
    
    # Показываем результат на экране
    Display.show_image(img)
    time.sleep(0.05)  # Небольшая задержка

# 5. Завершение работы
Display.deinit()
MediaManager.deinit()
```

## Важные моменты

- **tp.read(1)** возвращает список точек только при наличии касания
- **Необходимо использовать бесконечный цикл** для постоянного опроса сенсора
- **Важно обновлять экран** после каждого изменения изображения
- **Задержка в цикле** (`time.sleep()`) помогает снизить нагрузку на процессор
- **Координаты начинаются с левого верхнего угла** (0, 0)
  - X растёт вправо (0..639 для экрана 640px)
  - Y растёт вниз (0..479 для экрана 480px)
- **Слои отображения**: можно использовать несколько слоёв (LAYER_OSD2 и др.) для наложения графики

## Советы для начинающих

1. **Начинайте с примера `01_touch_basic.py`** - он самый простой и показывает основы
2. **Используйте `print()` для отладки** - выводите координаты и события в консоль
3. **Помните о системе координат**: X растёт вправо, Y растёт вниз от левого верхнего угла
4. **Для создания кнопок проверяйте попадание координат в прямоугольную область**:
   ```python
   if x_min < pt.x < x_max and y_min < pt.y < y_max:
       print("Кнопка нажата!")
   ```
5. **Используйте отдельные слои (overlay)** для временной графики - это упрощает анимацию
6. **Не забывайте очищать ресурсы** в конце программы: `Display.deinit()`, `MediaManager.deinit()`

## Рекомендации по запуску

1. Подключите плату Yahboom K230 к компьютеру через USB
2. Откройте CanMV IDE и подключитесь к плате
3. Выберите нужный пример (например, `01_touch_basic.py`)
4. Нажмите кнопку запуска (▶) в IDE
5. Коснитесь экрана, чтобы увидеть результат
