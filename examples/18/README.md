# Camera Display на K230

## Что это

Camera display — это вывод изображения с камеры на экран. Данный пример показывает три основных способа работы с камерой:
- Базовый вывод изображения на дисплей
- Наложение текста поверх видеопотока
- Сохранение кадров на SD-карту

## Как работает

```
камера → sensor.snapshot() → Display.show_image()
```

1. **Инициализация камеры**: Создаётся объект `Sensor`, настраивается разрешение и формат пикселей
2. **Инициализация дисплея**: Подключается дисплей ST7701 с заданным разрешением
3. **Запуск потока**: Вызывается `sensor.run()` для начала захвата изображения
4. **Основной цикл**: В бесконечном цикле получаем кадры и отображаем их на экране

## Примеры

| Файл | Описание | Особенности |
|------|----------|-------------|
| `01_camera_basic.py` | Базовый вывод | Простое отображение видео с камеры, подсчёт FPS |
| `02_camera_overlay.py` | Наложение текста | Добавление текста поверх изображения через `draw_string_advanced()` |
| `03_camera_capture_save.py` | Сохранение кадров | Периодическое сохранение кадров на SD-карту в формате JPG |

## Настройки по умолчанию

- **Формат пикселей**: RGB565 (16 бит на пиксель)
- **Разрешение**: 640×480 пикселей
- **Дисплей**: ST7701
- **Задержка между кадрами**: 5 мс

## Важно знать

- ⚠️ **`sensor.run()` обязателен** - без этого камера не начнёт захват изображения
- ⚠️ **`MediaManager.init()`** необходим для корректной работы медиапотоков
- ⚠️ Для сохранения файлов требуется установленная SD-карта
- ⚠️ Не забывайте вызывать `gc.collect()` для освобождения памяти

## Структура кода

Каждый скрипт содержит:
1. Шапку с описанием и авторством
2. Импорт необходимых библиотек
3. Настройку параметров (разрешение, формат)
4. Инициализацию оборудования
5. Основной цикл обработки кадров
