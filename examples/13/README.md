# AES примеры для K230

## Что такое AES

**AES (Advanced Encryption Standard)** — симметричный блочный шифр, который шифрует данные блоками по 16 байт (128 бит). Используется один и тот же ключ для шифрования и расшифровки.

### Основные свойства

- **Блочный шифр** — обрабатывает данные фиксированными блоками по 16 байт
- **Размеры ключа**: 16 байт (AES-128), 24 байта (AES-192), 32 байта (AES-256)
- **Симметричный** — один ключ для шифрования и расшифровки
- **Обратимая функция** — в отличие от хеш-функций (SHA), можно получить исходные данные

## Режимы работы AES

| Режим | Название | Описание | Безопасность |
|-------|----------|----------|--------------|
| **ECB** | Electronic Codebook | Каждый блок шифруется независимо | ❌ Небезопасен |
| **CBC** | Cipher Block Chaining | Каждый блок зависит от предыдущего | ✅ Безопасен |
| **CTR** | Counter | Потоковый режим на основе счётчика | ✅ Безопасен |
| **GCM** | Galois/Counter Mode | CTR + аутентификация данных | ✅ Наиболее безопасен |

### Минимальный код для шифрования

```python
import ucryptolib

key = b'1234567890abcdef'  # 16 байт = AES-128
plaintext = b'Data to encrypt!'  # Должно быть кратно 16 байтам

# Шифрование
cipher = ucryptolib.aes(key, ucryptolib.MODE_ECB)
encrypted = cipher.encrypt(plaintext)

# Расшифровка
cipher = ucryptolib.aes(key, ucryptolib.MODE_ECB)
decrypted = cipher.decrypt(encrypted)
```

## Важные замечания

### ⚠️ ECB режим небезопасен!

Режим **ECB (Electronic Codebook)** показывает одинаковые блоки открытого текста как одинаковые блоки шифротекста. Это позволяет злоумышленнику:
- Анализировать паттерны в зашифрованных данных
- Определять повторяющиеся элементы
- Проводить криптоанализ

**Используйте CBC, CTR или GCM режимы для реальных приложений!**

### Требования к данным

1. **Ключ**: должен быть ровно 16, 24 или 32 байта
2. **Данные**: длина должна быть кратна 16 байтам (для ECB и CBC)
3. **Для CBC режима**: требуется вектор инициализации (IV)

## Сравнение с SHA

| Характеристика | AES | SHA |
|----------------|-----|-----|
| Тип | Шифрование | Хеширование |
| Обратимость | ✅ Обратимо | ❌ Необратимо |
| Ключ | Требуется | Не требуется |
| Назначение | Защита конфиденциальности | Целостность данных |
