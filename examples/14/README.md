# Многопоточность на K230

## Что такое многопоточность

Многопоточность — это возможность выполнять несколько задач «одновременно» в рамках одной программы.

На плате **Yahboom K230** для работы с потоками используется встроенный модуль `_thread`.

## Основные понятия

- **Поток (thread)** — отдельный путь выполнения кода внутри программы
- **Главный поток** — основной поток, в котором запускается программа
- **Дочерние потоки** — дополнительные потоки, создаваемые главным потоком
- **Планировщик** — часть системы, которая решает, какой поток выполняется в данный момент

## Как это работает

1. Каждый поток выполняет свою функцию независимо
2. Потоки работают параллельно (логически, не обязательно физически)
3. Вывод сообщений в терминале может перемешиваться
4. Для корректной работы нужно давать планировщику шанс переключать потоки

## Важные правила

### 1. Используйте задержки (sleep)

```python
time.sleep(0.5)      # задержка 0.5 секунды
time.sleep_ms(500)   # задержка 500 миллисекунд
```

Без задержек поток может занять процессор полностью и другие потоки не смогут работать.

### 2. Синхронизируйте доступ к общим данным

Если несколько потоков работают с одной переменной, используйте **lock** (замок):

```python
lock = _thread.allocate_lock()  # создаём замок
lock.acquire()                   # захватываем замок
# ... работа с общими данными ...
lock.release()                   # освобождаем замок
```

## Основные функции модуля _thread

| Функция | Описание |
|---------|----------|
| `_thread.start_new_thread(func, args)` | Запускает новый поток с функцией `func` и аргументами `args` |
| `_thread.get_ident()` | Возвращает уникальный идентификатор текущего потока |
| `_thread.allocate_lock()` | Создаёт объект lock для синхронизации |

1. **Потоки выполняются одновременно** — вы видите вывод от обоих потоков
2. **Вывод перемешивается** — сообщения от разных потоков идут вперемешку
3. **Нужна синхронизация** — без lock данные могут повредиться

## Типичные ошибки новичков

❌ **Забыли sleep()** — поток занимает 100% процессора  
❌ **Нет lock при работе с общими данными** — возможны гонки данных  
❌ **Главный поток завершился** — программа упала, дочерние потоки остановлены  

## Итог

**Многопоточность** = возможность решать несколько задач одновременно

**Но помните:**
- Нужны задержки (`sleep`) для переключения потоков
- Нужна синхронизация (`lock`) для общих данных
- Главный поток должен работать, пока нужны дочерние потоки
