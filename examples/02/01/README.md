# FPIOA на K230

FPIOA (Field Programmable Input and Output Array) — это механизм переназначения функций выводов микроконтроллера. Проще говоря, один и тот же физический пин может работать как GPIO, UART, I2C, PWM и в других режимах. Это позволяет гибко настраивать периферию под вашу схему.

Этот каталог содержит простые примеры для новичков по работе с FPIOA на K230.

## Что такое FPIOA простыми словами

У микросхемы много внутренних модулей: UART, I2C, PWM, IIS и другие. Но наружу выведено меньше контактов, чем внутренних функций. Поэтому используется мультиплексирование: мы выбираем, какая именно функция будет подключена к конкретному пину.

Например, один и тот же вывод может быть:
- обычным GPIO;
- линией UART TX или RX;
- линией I2C SCL или SDA;
- выходом PWM.

Но одновременно у одного физического пина может быть только одна активная функция.

## Что важно понять новичку

1. Сначала нужно выбрать пины.
2. Потом назначить им функции через `FPIOA`.
3. После этого уже создавать и использовать нужный модуль: `UART`, `I2C`, `PWM` и так далее.

## Полезные пины на плате YAHBOOM K230

Согласно приложенному описанию распиновки:

- GPIO33 может использоваться как `GPIO33 / IIC0_SDA / IIS_WS / UART3_RXD`
- GPIO32 может использоваться как `GPIO32 / IIC0_SCL / IIS_CLK / UART3_TXD`
- GPIO34 может использоваться как `GPIO34 / IIC1_SCL / IIS_D_IN0 / PDM_IN3 / UART3_RTS`
- GPIO35 может использоваться как `GPIO35 / IIC1_SDA / IIS_D_OUT0 / PDM_IN1 / UART3_CTS`
- GPIO9 может использоваться как `UART1_TXD`
- GPIO10 может использоваться как `UART1_RXD`

В документе также показано, что для UART1 обычно используются GPIO9 и GPIO10, а пример инициализации выглядит так: `fpioa.set_function(9, fpioa.UART1_TXD, ie=0, oe=1)` и `fpioa.set_function(10, fpioa.UART1_RXD, ie=1, oe=0)` fileciteturn0file0.

## Важные замечания

- Перед назначением новой функции убедитесь, что пин не занят другим устройством.
- Не подключайте внешние устройства вслепую: сначала проверьте распиновку платы.
- Один и тот же пин нельзя одновременно использовать, например, и как UART, и как GPIO.
- Если пример не запускается, начните с вызова `fpioa.help()` и проверьте доступные функции для нужного пина.
