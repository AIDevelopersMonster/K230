# REPL на K230 / CanMV

Этот документ объясняет, как подключиться к **REPL-консоли** на плате K230 / CanMV и как работать с ней в разных терминальных программах, включая **MobaXterm** и **PuTTY**.

---

## Что такое REPL

**REPL** — это интерактивная консоль MicroPython.

Название расшифровывается так:

- **R**ead — прочитать команду
- **E**val — выполнить её
- **P**rint — вывести результат
- **L**oop — ждать следующую команду

Через REPL можно:

- быстро проверять команды
- импортировать модули
- тестировать GPIO, UART, камеру и другие функции
- смотреть ошибки прямо в консоли
- перезапускать скрипты и вручную выполнять код

---

## Что понадобится

Перед подключением убедитесь, что у вас есть:

- плата **K230 / CanMV**
- USB-кабель для подключения к ПК
- установленный USB-UART драйвер, если он нужен вашей системе
- терминальная программа:
  - CanMV IDE
  - MobaXterm
  - PuTTY
  - Tera Term
  - minicom / screen / picocom в Linux

---

## Основная идея подключения

После подключения платы к компьютеру обычно появляется **последовательный порт**:

- в Windows это обычно `COM3`, `COM4`, `COM5` и т.д.
- в Linux это обычно `/dev/ttyUSB0`, `/dev/ttyACM0` и т.д.
- в macOS это обычно `/dev/tty.usbserial-*` или `/dev/tty.usbmodem-*`

Далее вы открываете этот порт в терминальной программе и работаете с REPL.

---

## Рекомендуемые параметры порта

Если конкретная прошивка не требует иного, обычно используют:

- **Baud rate:** `115200`
- **Data bits:** `8`
- **Parity:** `None`
- **Stop bits:** `1`
- **Flow control:** `None`

Это классическая схема **115200 8N1**.

> Если консоль не отвечает или выводит нечитаемые символы, первым делом проверьте скорость порта.

## Подключение через CanMV IDE

> ⚠️ По текущему опыту для этой платы подключение через **CanMV IDE** может не работать или работать нестабильно. Если REPL не открывается в IDE, используйте **MobaXterm** или **PuTTY**.

Если вы всё же хотите попробовать CanMV IDE:

1. Подключите плату по USB.
2. Откройте **CanMV IDE**.
3. Выберите правильный COM-порт.
4. Откройте последовательную консоль / терминал.
5. Нажмите **Enter**.

Если приглашение REPL не появляется, попробуйте:

- отключить и снова подключить плату
- закрыть IDE и открыть её заново
- проверить, не занят ли COM-порт другой программой
- перейти на **MobaXterm** или **PuTTY**

Пример приглашения:

```python
>>>
```

## Подключение через MobaXterm

**MobaXterm** удобен тем, что объединяет SSH, serial и другие виды соединений в одном приложении.

### Шаги

1. Подключите K230 к компьютеру.
2. Откройте **MobaXterm**.
3. Нажмите **Session**.
4. Выберите **Serial**.
5. В поле **Serial port** выберите нужный COM-порт.
   - Например: `COM5`
6. В поле **Speed** укажите:
   - `115200`
7. Нажмите **OK**.
8. После открытия терминала нажмите **Ctrl + C**.
9. Затем нажмите **Enter**.

Если всё подключено правильно, вы увидите REPL-приглашение:

```python
>>>
```

### Важно

По текущему опыту на этой плате в **MobaXterm** REPL может появляться **только после `Ctrl + C`**. Это нормально: сочетание клавиш прерывает текущий код или выводит устройство обратно в интерактивную консоль.

### Если ничего не видно

Проверьте:

- правильный ли выбран COM-порт
- не открыт ли этот порт в другой программе
- совпадает ли скорость порта
- исправен ли USB-кабель
- нажали ли вы **Ctrl + C**, а потом **Enter**

## Подключение через PuTTY

**PuTTY** — один из самых популярных терминалов для Windows.

### Шаги

1. Подключите плату K230 по USB.
2. Запустите **PuTTY**.
3. В разделе **Connection type** выберите **Serial**.
4. В поле **Serial line** укажите COM-порт:
   - например `COM5`
5. В поле **Speed** укажите:
   - `115200`
6. Нажмите **Open**.
7. После открытия окна терминала нажмите **Ctrl + C**.
8. Затем нажмите **Enter**.

Если подключение успешно, появится:

```python
>>>
```

### Важно

По текущему опыту на этой плате в **PuTTY** REPL также может появляться **только после `Ctrl + C`**.

### Полезные замечания для PuTTY

- Если видите пустой экран — сначала нажмите **Ctrl + C**, затем **Enter**.
- Если появляются «кракозябры», почти всегда проблема в неправильной скорости.
- Если терминал сразу закрывается — проверьте, существует ли COM-порт и подключена ли плата.

## Подключение в Linux

В Linux можно использовать `screen`, `minicom` или `picocom`.

### Пример для screen

```bash
screen /dev/ttyUSB0 115200
```

После подключения, если приглашение REPL не видно, попробуйте:

```text
Ctrl + C
```

а затем нажмите `Enter`.

### Пример для picocom

```bash
picocom -b 115200 /dev/ttyUSB0
```

### Пример для minicom

Сначала настройте порт, затем откройте сессию.

Если у вас нет прав доступа к последовательному порту, добавьте пользователя в нужную группу, например `dialout`.

---

## Подключение в macOS

Сначала найдите устройство:

```bash
ls /dev/tty.*
```

Далее можно подключиться так:

```bash
screen /dev/tty.usbserial-XXXX 115200
```

или:

```bash
screen /dev/tty.usbmodem-XXXX 115200
```

Если REPL сразу не появился, попробуйте `Ctrl + C`, затем `Enter`.

## Как понять, что REPL работает

Если REPL запущен нормально, вы можете выполнить простые команды:

```python
print(123)
```

или:

```python
2 + 3
```

Ожидаемый результат:

```python
123
5
```

Также можно проверить версию:

```python
import sys
sys.version
```

## Мягкий перезапуск REPL

В MicroPython обычно используется сочетание:

- **Ctrl + D** — soft reset

Это перезапускает интерпретатор, но обычно не эквивалентно полному выключению питания.

После мягкого перезапуска REPL снова появляется приглашение.

---

## Прерывание выполняющегося кода

Если программа зациклилась и вы хотите остановить её из терминала:

- **Ctrl + C** — прервать выполнение текущего скрипта

Это очень полезно, если запущен бесконечный цикл:

```python
while True:
    print("loop")
```

На практике для этой платы `Ctrl + C` также помогает вернуть устройство в REPL при подключении через некоторые терминалы.

---

## Если REPL не появляется

Проверьте по порядку:

1. Плата действительно включена
2. USB-кабель передаёт данные, а не только питание
3. Установлен нужный драйвер USB-UART
4. Выбран правильный COM-порт
5. Скорость стоит `115200`
6. Порт не занят другой программой
7. После подключения нажмите **Ctrl + C**, затем **Enter**

---

## Типичные проблемы

### Пустой экран

Причины:

- неверный COM-порт
- плата не загрузилась
- терминал не отправил Enter
- устройство ещё не вышло в REPL

Решение:

- нажмите **Ctrl + C**, потом **Enter**

### Нечитаемые символы

Причины:

- неправильная скорость порта

Решение:

- проверьте `115200`

### Порт не открывается

Причины:

- порт занят другой программой
- нет драйвера
- кабель неисправен

### Код не реагирует

Попробуйте:

- нажать **Ctrl + C**
- затем **Enter**
- затем **Ctrl + D**

## Когда использовать REPL, а когда запускать файл

### REPL удобен для:

- коротких проверок
- тестирования команд
- отладки модулей
- исследования API

### Файлы `.py` удобнее для:

- полноценных программ
- примеров
- автозапуска
- хранения проекта

На практике обычно делают так:

1. пробуют команды в REPL
2. затем переносят рабочий код в `.py` файл

---

## Рекомендуемый рабочий процесс

1. Подключить плату
2. Открыть REPL
3. Если консоль пустая — нажать **Ctrl + C**, затем **Enter**
4. Проверить `print("hello")`
5. Импортировать нужный модуль
6. Проверить простую функцию
7. Перенести рабочий код в отдельный файл

---

## Краткая памятка

- **Enter** — показать приглашение REPL
- **Ctrl + C** — остановить текущий код или вернуть устройство в REPL
- **Ctrl + D** — мягко перезапустить интерпретатор
- **115200 8N1** — типовая настройка порта

---

## Итог

REPL — это самый быстрый способ начать работу с K230 / CanMV.

Через него удобно:

- проверять MicroPython-команды
- изучать модули
- отлаживать подключение
- тестировать периферию

Для Windows в текущей практике надёжнее использовать **MobaXterm** или **PuTTY**, причём после подключения часто помогает последовательность:

1. **Ctrl + C**
2. **Enter**

Если CanMV IDE не показывает REPL, лучше сразу перейти на внешний терминал.
