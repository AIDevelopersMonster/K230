# SHA-256 примеры для K230

## 📚 Что такое SHA-256

**SHA-256** — это криптографическая хеш-функция из семейства SHA-2.
Она принимает входные данные любой длины и возвращает результат фиксированной длины:

- **256 бит**
- **32 байта**
- **64 hex-символа** при текстовом представлении

> ⚠️ **Важно:** SHA-256 — это не шифрование. Исходные данные из хеша восстановить нельзя практическим способом.

## 🔑 Основные свойства SHA-256

| Свойство | Описание |
|----------|----------|
| **Детерминированность** | Одинаковые входные данные всегда дают одинаковый хеш |
| **Эффект лавины** | Изменение даже одного бита сильно меняет итоговый хеш (примерно 50% бит) |
| **Фиксированная длина** | Независимо от размера входа digest всегда 32 байта |
| **Быстрота** | Вычисление хеша происходит очень быстро |
| **Односторонность** | По хешу нельзя восстановить исходные данные |

## 💡 Как SHA-256 используется на K230

На K230 SHA-256 удобно использовать для:

- ✅ проверки целостности файлов на SD-карте;
- ✅ контроля изменений конфигурации;
- ✅ проверки корректности передачи данных по UART, Wi-Fi или сети;
- ✅ подготовки данных для цифровой подписи и верификации;
- ✅ учебных экспериментов по криптографии и безопасности;
- ✅ создания уникальных идентификаторов для данных.

## ⚙️ Важная особенность CanMV / K230

На некоторых прошивках K230 объект `hashlib.sha256(...)` может не поддерживать метод `hexdigest()`.
Поэтому в этих примерах используется безопасный для K230 вариант:

```python
import hashlib
import binascii

digest = hashlib.sha256(data).digest()  # получаем 32 байта
hex_text = binascii.hexlify(digest)     # преобразуем в hex-строку
```

Если `hexlify()` возвращает `bytes`, можно перевести их в строку через `.decode()`.

## 📖 Что показывают примеры

### 1️⃣ Базовый пример (`01_sha256_basic.py`)
Показывает разницу между бинарным `digest()` и hex-представлением этого digest.
Вы увидите:
- как вычислить хеш строки;
- что одинаковые данные дают одинаковый хеш;
- как выглядят 32 байта хеша в разных форматах.

### 2️⃣ Проверка эталонных значений (`02_sha256_reference_test.py`)
Показывает, что SHA-256 можно считать по частям через `update()` и что результат совпадает с заранее известным значением.
Вы научитесь:
- передавать данные порциями;
- проверять корректность работы хеш-функции;
- работать с большими объёмами данных.

### 3️⃣ Эффект лавины (`03_sha256_avalanche.py`)
Показывает, что если изменить всего один символ, итоговый хеш меняется очень сильно (примерно 128 бит из 256).
Это демонстрирует криптографическую стойкость SHA-256.

## ❓ Почему SHA-256 не рекомендуют для паролей в чистом виде

SHA-256 очень быстрый. Именно поэтому он удобен для проверки целостности файлов, но неидеален для хранения паролей.
Злоумышленник может быстро перебрать миллионы вариантов паролей.

Для паролей обычно используют специальные функции:

| Функция | Особенность |
|---------|-------------|
| **Argon2** | Победитель конкурса Password Hashing Competition, устойчив к GPU-атакам |
| **bcrypt** | Использует соль и адаптивную стоимость |
| **scrypt** | Требует много памяти, что затрудняет аппаратные атаки |
| **PBKDF2** | Многократное применение хеш-функции |

## 🔍 Примеры использования в реальных проектах

```python
# Проверка целостности файла
def check_file_integrity(filename, expected_hash):
    with open(filename, 'rb') as f:
        data = f.read()
    actual_hash = hashlib.sha256(data).digest()
    return actual_hash == expected_hash

# Создание уникального ID для данных
def create_data_id(data):
    return hashlib.sha256(data).digest()
```
