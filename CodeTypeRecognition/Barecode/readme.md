# K230: Barcode Recognition

Эта папка содержит демонстрационные примеры для **Yahboom K230 Vision Module** по теме распознавания 1D штрих-кодов. Камера K230 получает изображение, ищет штрих-коды через `img.find_barcodes()`, определяет тип кода, извлекает полезную нагрузку `payload` и показывает результат на экране.

> В названии папки используется `Barecode`, как было указано в задании. Правильный английский термин — `Barcode`.

## Файлы

| Файл | Назначение |
| --- | --- |
| `01_barcode_recognition.py` | Базовая демонстрация: поиск штрих-кода, рамка, тип, payload и FPS. |
| `02_barcode_uart.py` | Распознавание штрих-кода + отправка данных через UART/Yahboom protocol. |
| `03_barcode_file_io.py` | Распознавание штрих-кода + чтение ROI из файла и запись результата в файл. |
| `readme.md` | Инструкция на русском языке. |
| `readme_en.md` | Instruction in English. |

## Быстрый старт

1. Сгенерируйте 1D штрих-код. Можно использовать barcode generator из материалов Yahboom или любой онлайн-генератор.
2. Откройте `01_barcode_recognition.py` в CanMV IDE.
3. Подключите K230 к компьютеру по USB.
4. Запустите скрипт.
5. Наведите камеру на штрих-код так, чтобы он был в фокусе.
6. На экране появится рамка вокруг штрих-кода, тип кода и расшифрованный текст.
7. В консоли CanMV IDE будет выводиться строка вида:

```text
Barcode CODE128, Payload "What can I say"
```

## Как работает распознавание штрих-кодов

Основная функция:

```python
codes = img.find_barcodes()
```

Она возвращает список объектов `image.barcode`. Для каждого найденного штрих-кода можно получить:

```python
code.rect()      # (x, y, w, h), рамка штрих-кода
code.payload()   # текст или число внутри штрих-кода
code.type()      # тип штрих-кода
code.rotation()  # угол поворота
code.quality()   # качество/число успешных обнаружений
```

Рамка рисуется так:

```python
img.draw_rectangle(code.rect(), thickness=6, color=(46, 47, 48))
```

Текст выводится так:

```python
img.draw_string_advanced(10, 10, 28, "Data: " + payload, color=(255, 255, 255))
```

## Поддерживаемые типы штрих-кодов

В примерах добавлен словарь `BARCODE_TYPES`, который переводит числовой тип в понятное имя:

```python
BARCODE_TYPES = {
    image.EAN13: "EAN13",
    image.ISBN13: "ISBN13",
    image.CODE39: "CODE39",
    image.CODE93: "CODE93",
    image.CODE128: "CODE128",
}
```

Полный список в коде включает EAN2, EAN5, EAN8, UPCE, ISBN10, UPCA, EAN13, ISBN13, I25, DATABAR, CODABAR, CODE39, PDF417, CODE93 и CODE128.

## ROI: ускорение поиска

По умолчанию пример ищет штрих-коды по всему кадру:

```python
BARCODE_ROI = None
```

Для ускорения можно ограничить область поиска, например горизонтальной полосой в центре кадра:

```python
BARCODE_ROI = (0, 200, 640, 80)
```

ROI имеет формат:

```python
(x, y, w, h)
```

Штрих-коды — линейные изображения, поэтому часто достаточно сканировать узкую область 40, 80 или 160 пикселей по высоте, если код находится в этой зоне.

## Пример с UART

Файл `02_barcode_uart.py` отправляет данные через Yahboom protocol:

```python
pto_data = pto.get_barcode_data(x, y, w, h, payload)
uart.send(pto_data)
```

UART инициализируется так:

```python
uart = YbUart(baudrate=115200)
```

Если в прошивке нет библиотек `libs.YbProtocol` или `ybUtils.YbUart`, пример не падает, а отключает UART и продолжает распознавание на экране.

## Чтение файлов на K230

Файл `03_barcode_file_io.py` показывает чтение настроек из текстового файла.

Имя файла:

```text
barcode_config.txt
```

Пример содержимого:

```text
# K230 Barcode Recognition config
USE_ROI=0
ROI=0,200,640,80
```

Чтение файла:

```python
with open("/sdcard/barcode_config.txt", "r") as f:
    text = f.read()
```

Разбор строки:

```python
key, value = line.split("=", 1)
```

Разбор ROI:

```python
parts = value.replace(" ", "").split(",")
roi = tuple([int(v) for v in parts])
```

На CanMV/K230 относительный путь иногда недоступен для записи при запуске из IDE. Поэтому пример сначала ищет доступную папку:

```python
FILE_DIR_CANDIDATES = ["/sdcard", "/data", "/flash", "/"]
```

Если файловая система недоступна для записи, пример продолжает работать со значениями по умолчанию.

## Запись файлов на K230

Запись выполняется через `open(..., "w")`:

```python
with open("/sdcard/barcode_result.txt", "w") as f:
    f.write("payload=What can I say\n")
```

Режимы открытия:

| Режим | Что делает |
| --- | --- |
| `"r"` | Чтение существующего файла. |
| `"w"` | Запись с заменой старого содержимого. Если файла нет, он будет создан. |
| `"a"` | Добавление текста в конец файла. |

Файл результата `barcode_result.txt` обновляется при распознавании:

```text
time_ms=123456
fps=32.500
type=CODE128
payload=What can I say
x=102
y=180
w=420
h=90
```

Почему не записывать каждый кадр: штрих-код может распознаваться много раз подряд, поэтому в примере есть ограничение частоты записи.

## Рекомендации для хорошего распознавания

1. Наведите резкость на штрих-код.
2. Уберите блики с экрана или бумаги.
3. Разместите штрих-код крупно в кадре.
4. Используйте хороший контраст: черные полосы на белом фоне.
5. Если кадр тормозит, используйте ROI.
6. Не используйте сжатые или Bayer-изображения для `find_barcodes()`.
7. Если код не читается, попробуйте повернуть камеру или увеличить размер штрих-кода.
