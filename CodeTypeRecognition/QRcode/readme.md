# K230: QR Code Recognition

Эта папка содержит демонстрационные примеры для **Yahboom K230 Vision Module** по теме распознавания QR-кодов. Камера K230 получает изображение, ищет QR-коды через `img.find_qrcodes()`, извлекает полезную нагрузку `payload` и показывает результат на экране.

QR-код отличается от обычного штрих-кода тем, что хранит данные в двумерной матрице. Поэтому QR-коды обычно вмещают больше информации, могут содержать URL, текст или другие данные и поддерживают уровни коррекции ошибок.

## Файлы

| Файл | Назначение |
| --- | --- |
| `01_qrcode_recognition.py` | Базовая демонстрация: поиск QR-кода, рамка, payload и FPS. |

| `readme_en.md` | Instruction in English. |

## Быстрый старт

1. Сгенерируйте QR-код. Можно использовать QR Code Generator из материалов Yahboom или любой онлайн-генератор.
2. Откройте `01_qrcode_recognition.py` в CanMV IDE.
3. Подключите K230 к компьютеру по USB.
4. Запустите скрипт.
5. Наведите камеру на QR-код так, чтобы он был крупным, ровным и в фокусе.
6. На экране появится рамка вокруг QR-кода и его содержимое.
7. В консоли CanMV IDE будет выводиться payload QR-кода.

## Как работает распознавание QR-кодов

Основная функция:

```python
qr_codes = img.find_qrcodes()
```

Она возвращает список объектов `image.qrcode`. Для каждого найденного QR-кода можно получить:

```python
qr.rect()          # (x, y, w, h), рамка QR-кода
qr.payload()       # текст, URL или другие данные внутри QR-кода
qr.version()       # версия QR-кода
qr.ecc_level()     # уровень коррекции ошибок
qr.mask()          # маска QR-кода
qr.data_type()     # тип данных
qr.eci()           # индикатор кодировки
qr.is_numeric()    # True, если данные числовые
qr.is_alphanumeric() # True, если данные буквенно-цифровые
qr.is_binary()     # True, если данные бинарные
qr.is_kanji()      # True, если данные Kanji
```

Рамка рисуется так:

```python
img.draw_rectangle(qr.rect(), thickness=3, color=(200, 0, 0))
```

Payload выводится на экран так:

```python
img.draw_string_advanced(0, 0, 26, "QR: " + qr.payload(), color=(255, 255, 255))
```

## ROI: ускорение поиска

По умолчанию пример ищет QR-коды по всему кадру:

```python
QRCODE_ROI = None
```

Для ускорения можно ограничить область поиска:

```python
QRCODE_ROI = (120, 60, 400, 360)
```

ROI имеет формат:

```python
(x, y, w, h)
```

Если QR-код всегда находится в центре кадра, ROI может повысить FPS.

## Пример с UART

Файл `02_qrcode_uart.py` отправляет данные через Yahboom protocol:

```python
pto_data = pto.get_qrcode_data(x, y, w, h, payload)
uart.send(pto_data)
```

UART инициализируется так:

```python
uart = YbUart(baudrate=115200)
```

Если в прошивке нет библиотек `libs.YbProtocol` или `ybUtils.YbUart`, пример не падает, а отключает UART и продолжает распознавание на экране.

## Чтение файлов на K230

Файл `03_qrcode_file_io.py` показывает чтение настроек из текстового файла.

Имя файла:

```text
qrcode_config.txt
```

Пример содержимого:

```text
# K230 QR Code Recognition config
USE_ROI=0
ROI=120,60,400,360
```

Чтение файла:

```python
with open("/sdcard/qrcode_config.txt", "r") as f:
    text = f.read()
```

Разбор строки:

```python
key, value = line.split("=", 1)
```

Разбор ROI:

```python
parts = value.replace(" ", "").split(",")
roi = tuple([int(v) for v in parts])
```

На CanMV/K230 относительный путь иногда недоступен для записи при запуске из IDE. Поэтому пример сначала ищет доступную папку:

```python
FILE_DIR_CANDIDATES = ["/sdcard", "/data", "/flash", "/"]
```

Если файловая система недоступна для записи, пример продолжает работать со значениями по умолчанию.

## Запись файлов на K230

Запись выполняется через `open(..., "w")`:

```python
with open("/sdcard/qrcode_result.txt", "w") as f:
    f.write("payload=Yahboom Technology Co.,Ltd\n")
```

Режимы открытия:

| Режим | Что делает |
| --- | --- |
| `"r"` | Чтение существующего файла. |
| `"w"` | Запись с заменой старого содержимого. Если файла нет, он будет создан. |
| `"a"` | Добавление текста в конец файла. |

Файл результата `qrcode_result.txt` обновляется при распознавании:

```text
time_ms=123456
fps=28.500
payload=Yahboom Technology Co.,Ltd
x=180
y=90
w=240
h=240
version=3
ecc_level=3
data_type=2
```

Почему не записывать каждый кадр: QR-код может распознаваться много раз подряд, поэтому в примере есть ограничение частоты записи.

## Уровень коррекции ошибок QR-кода

QR-коды поддерживают уровни коррекции ошибок L, M, Q и H. Чем выше уровень, тем больше повреждений или помех может выдержать QR-код, но тем плотнее становится рисунок. Для демонстраций удобно использовать высокий уровень, например H, если QR-код может быть частично закрыт или зашумлен.

## Рекомендации для хорошего распознавания

1. Держите QR-код в фокусе.
2. Не допускайте сильного наклона и перспективных искажений.
3. Уберите блики с экрана или бумаги.
4. Разместите QR-код крупно в кадре.
5. Используйте хороший контраст: черный QR-код на белом фоне.
6. Если кадр тормозит, используйте ROI.
7. Если код не читается, попробуйте увеличить размер QR-кода или улучшить освещение.
